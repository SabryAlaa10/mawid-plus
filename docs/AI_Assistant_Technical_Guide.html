دليل مساعد موعد+ التقني

مسار كامل للكود الفعلي في Ai_Mawid/mawid-assistant — تحديث حسب قراءة الملفات للمستودع.

1. نظرة عامة على النظام

ما هو الـ AI Assistant؟

خدمة FastAPI (app/main.py) تتلقّى طلبات JSON على /api/chat وغيرها، وتتحقق من هوية المريض عبر Bearer JWT من Supabase. تستدعي نموذج لغة خارجيًا (مسارّان: OpenRouter ثم Groq في services/llm.py L132–144) وتعيد حقولًا منظمّة لتوجيه تجربة المستخدم داخل موعد+.

المشكلة التي يحلها

محادثة عربية تساعد في التوجيه نحو التخصص المناسب دون تشخيص دوائي، ثم تحويل ذلك إلى أسطر في جدول doctors مرتبة بالتقييم (get_doctors_by_specialty، database.py L51–65). كما تدعم خدمات مساندة لجلب الفترات الزمنية والحجز.

الموقع في Mawid+

Supabase يبقى المصدر الرسمي للبيانات. التطبيق يرسل للمساعد نفس رمز الجلسة الذي يثبت فيه المريض عبر GoTrue، بينما السيرفر يستخدم service role لقراءة/كتابة قاعدة البيانات بصلاحيات أعلى من RLS للعميل.

def get_doctors_by_specialty(specialty: str, limit: int = 3) -> dict:
    response = (
        supabase.table("doctors")
        .select("*")
        .ilike("specialty", f"%{specialty}%")
        .order("rating", desc=True)
        .limit(limit)
        .execute()
    )
    if not response.data:
        return {"doctors": [], "matched": False}
    return {"doctors": response.data, "matched": True}

10. نظام RAG — rag.py

RAG: استرجاع مقاطع قريبة دلالياً لتغذية الصياغة المرجعية للنموذج — يقلّل ابتذال التطمينات ويربط الرد بتوجيه عام آمن موجود في المتجر.

النموذج الثابت: paraphrase-multilingual-MiniLM-L12-v2 (بعد المتجهات 384) L10؛ يحمّل مرة واحدة عند استيراد الوحدة (L11).

search_medical_knowledge L69 يستدعي RPC اسمها match_kb_chunks مع threshold 0.35.

_is_trivial_message: قصيرة جداً أو تحيات جاهزة أو أرقام فقط ⇒ تخطي البحث (L79–88).

بعد النتائج، قد يُزال قسم جلدي إذا ظهر سياق قلبي وفق تلميحات الدورة الدموية (L51–67).

except Exception as e:
    logger.warning(f"RAG failed... {e}")
    return ""

أي خطأ دلالي أو شبكة ⇒ يستمر LLM بدون سياق — لا يقع السيرفر.

16. النموذج الأماني ثلاثي الطبقات

  • المصادقة: معظم نقاط عمل الأعمال لا تخدم بدون مستخدم حقيقي.
  • CORS: قائمة بدل الزائدة السابقة — يقل خطر المتصفحات، لا يقي الأدوات المحلية.
  • خدمة المرور الموثقة: يفتح حقول Postgres شاملة — يقتضي حافة شبكة آمنة وإدارة سريّة ولوجيات آمنة.

غير مصادّق يُسمح له: فقط نقاط صحية لا تكشّف هوية؛ حالياً /health.

مخاطر متبقية: تسرّب JWT في سجلات، كثافة الموارد لمكتبة pytorch/sentence_transformers الهجينة، ومطاببة عناوين LLM بخصوصية طبّية.

18. أسئلة مقابلة استلام وأجوبة مختصرة

١٥ أسئلة — استخدم هذا القسم كنقاط حفظ سريعة.

1. لماذا يستخدم الخادم service role وليس مفتاح anon فقط؟

لمطالبة جداول الأطباء بترتيب التقييم دون تقييد RLS للعميل، ولتفعيل دوال RPC مثل match_kb_chunks وحجوزات موثوقة؛ راجع تحذير database.py L21–23.

2. كيف تُحمى مسارات /api/chat؟

عبر Depends(get_current_user) في تعريف المسار (chat.py L40)، ما يفرض رأس Authorization: Bearer … صالحًا.

3. لماذا مساران للتخصص (LLM ثم مطابقة DB)؟

النموذج قد يخرج تسمية حرة؛ match_specialty_to_db يضمن محاذاة مع قيم doctors.specialty الفعلية قبل ilike.

4. ماذا إن لم يُضبط OPENROUTER ولا GROQ؟

call_llm يرفع استثناء (llm.py L144)؛ chat.py يلتقط أي خطأ في كتلة RAG+LLM ويعيد رسالة عربية آمنة (L47–64).

5. ما الفرق بين hist وcomposite؟

_history_for_llm يغذي أدوار user/assistant؛ _composite_user_text يجمع نص المريض فقط لاشتقاق كلمات مفتاحية من كلامه المباشر (chat.py L20–28، L31–36).

6. متى يكون specialty_available=True؟

عندما ready_for_doctors ووجد تخصص مطابق وفتح get_doctors_by_specialty عددًا >0 مع matched (chat.py L95–97).

7. لماذا لا تُحمّل قائمة الأطباء رغم ready_for_doctors؟

فراغ التخصص، أو match_specialty_to_db يعيد None، أو الاستعلام لا يجد صفًا؛ فيُضاف النص العربي الطويل بعد الرسالة (L99–100).

8. لماذا get_close_matches(..., cutoff=0.6)؟

لتقريب أخطاء مطبعية أو اختلاف بسيط في صياغة العربية بين تصريح اللغوي وقيم عمود المتجر.

9. لماذا تصفية مقاطع «جلدية» أحيانًا في RAG؟

إذا حمل الاستعلام المركّب تلميحات دورة دموية ولا يظهر طفح جلدي؛ يهدف لتقليل سياق جلدي مضلل (rag.py L51–67).

10. متى يُستخدم JWT محلي؟

إذا لم يؤدِ تحقّق Supabase إلى مستخدم بينما يوجد SUPABASE_JWT_SECRET (auth.py L105–106).

11. لماذا OpenRouter يدوّر على أكثر من model؟

للمحاولة التالية بعد 429 أو أخطاء لكل نموذج (llm.py L157–185)، ثم رسالة خطأ ثابتة كـ JSON.

12. لماذا لا سقوط خام للعملاء على أخطاء LLM؟

تهيئة واجهة المستخدم تفضل ردًا عربيًا مهذبًا دون traceback في المسار العام للمحادثة.

13. كيف يُنشأ رقم الطابور؟

عد المواعيد غير الملغاة لذلك اليوم والطبيب +1 قبل الإدراج (database.py L131–141).

14. لماذا مثال الـ Prompt يذكر «الأمراض العصبية»؟

هو جزء ثابت من SYSTEM_PROMPT (llm.py L57–58) وقد لا يتوافق حرفيًا مع قيم المتجر؛ المنطق اللاحق يحاول المطابقة الديناميكية والكلمات المفتاحية تُقيّد التسميات.

15. ما وظيفة services/booking.py؟

ملف تجميعي يعيد تصدير book_appointment وget_available_slots من database.py للتوسعة المستقبلية.
